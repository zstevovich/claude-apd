02 / 25
Problem

LLM piše kod.
Ko proverava da je pipeline ispoštovan?

Svaki tim "ima proces":
spec → plan → implementacija → review → testovi → commit.
U realnosti, kada LLM radi sam — proces je honor system.
Nema mehanizma da zaustavi agenta koji:
  • preskoči spec i krene direktno u kod
  • piše tamo gde mu nije dozvoljeno
  • commituje bez testova i bez review-a
  • izmeni .done fajl i pravi se da je sve uredu
CENTRALNA HIPOTEZA

Bez mehaničkog

enforcementa, proces je

sugestija.

Konvencija koju slabiji model u lošijem dan-u neće poštovati — i ti to nećeš primetiti dok ne pukne u produkciji.

Lean vs Full

  • Full (default): svih 5 gate-ova + adversarial pass
  • Lean: preskače adversarial kada je spec ≤2 R-kriterija
  • Lean opt-out je mehanički: adversarial: skip — <reason> u spec-card
