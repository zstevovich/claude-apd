# v6.6 — Inline reviewer (opt-in) + telemetry

> Pisano u v6.5.4 sesiji. Izvršiti kao zaseban minor bump posle što baseline
> v6.5.x cluster slegne. Ne ulazi u patch chain.

## Cilj

Uvesti opt-in mod gde orkestrator radi prvi review **inline** (sa novim
`apd-review` skill-om), umesto da dispatch-uje `code-reviewer` kao
zaseban subagent. Adversarial-reviewer ostaje **uvek dispatched** —
strukturni anchor protiv simulacije.

Motivacija:
- Dispatched `code-reviewer` košta 3–8 min po task-u (start + load + Read svih
  fajlova orkestrator je već pročitao + write reviewer.done).
- Orkestrator ima punu kontekstualnu sliku: spec, plan, builder dispatch
  intent, prior fail-ovi, korisnikova intencija. Reviewer dispatched ne
  vidi ništa od toga osim svog prompt-a.
- Codex već radi inline builder (`BUILDER_RAN="codex-orchestrator"` u
  `pipeline-advance:321-322`). Simetrija: inline reviewer u CC.

Glavni rizik: orkestrator simuliraсе review (piše `.reviewer-trace` bez
stvarne pažnje), kao u `feedback-adversarial-summary-bypass`. Adversarial-D
telemetry je empirijski safety net: ako D-count jumpuje na inline pipeline-ima
nasprema dispatched-im, automatski hard-block i forsiranje dispatched.

## Plan migracije — fazno

### Phase 1 — skill + state + spec-card field

**Deliverable:** opt-in mehanizam postoji, ne radi ništa funkcionalno još.

1. `skills/apd-review/SKILL.md` (CC) + `plugins/apd/skills/apd-review/{SKILL.md, agents/openai.yaml}` (Codex).
   - Strukturisani output: za svaki R-criterion `## R<N>: PASS|FAIL|N/A` + obavezan blok `Citations:` sa `file.ext:LN — kratko obrazloženje` (bar 1 per R).
   - Mandatorni Read calls pre write-a `.reviewer-trace`: svi fajlovi pomenuti u `builder.done` + `implementation-plan.md` agents block.
   - Iron Law (template-style): "Ako citiraš line number koji ne postoji, fail-uješ ceo review. Ne pretpostavljaj — otvori fajl."
   - Anti-patterns: rubber-stamp PASS, generic "looks good", PASS bez Citations bloka, copy-paste R-criteria iz spec-card kao "obrazloženje".

2. `spec-card.md` parser u `pipeline-advance`: novo polje `reviewer: orchestrator | dispatched`. Default `dispatched` (no behavioral change). Bold/italic tolerance kao kod `adversarial: max_defects` (v6.1.2).

3. `.reviewer-mode` state file u `.apd/pipeline/`: sadrži `dispatched|orchestrator` u trenutku spec advance-a. Imutabilan tokom task-a (kao `.spec-max-defects-history` iz v6.3 D). Re-advance briše, novi spec piše ponovo.

### Phase 2 — pipeline-advance reviewer branch

**Deliverable:** kada je `reviewer: orchestrator`, reviewer step verifikuje `.reviewer-trace` umesto `|stop|code-reviewer|` u `.agents`.

`pipeline-advance reviewer` granu na osnovu `.reviewer-mode`:

**Dispatched grana** (current behavior):
- Verifikuje `|stop|<reviewer-name>|` u `.agents` log-u (lines ~445-490).
- Validate-agent HMAC.
- Suspicious + blocked agents check.
- Sve isto kao danas.

**Orchestrator grana** (NEW):
- Hard-block ako `.apd/pipeline/.reviewer-trace` ne postoji.
- Parse:
  - Header line: `SHA-BUILDER: <sha256>` i `SHA-SPEC: <sha256>`. Hard-block ako se ne podudaraju sa stvarnim `sha256(builder.done) + sha256(spec-card.md)`.
  - Za svaki R-criterion iz spec-card.md: mora postojati `## R<N>: PASS|FAIL|N/A`. Manjak = hard-block.
  - Za svaki PASS: minimum 1 `file.ext:LN` referenca u Citations bloku.
- File:line validation:
  - Path mora biti u builder scope-u (parse iz `implementation-plan.md` `### Agents` block ili `.agents` builder stop event-ova).
  - File postoji (relative to `$PROJECT_DIR`).
  - LN <= `wc -l < file`.
  - Sve hard-block na fail.
- Write `reviewer.done` sa istim formatom kao dispatched grana.
- Set `.adversarial-pending` (isto kao danas).
- Increment `.reviewer-count` (cycle cap iz v6.3 B važi i ovde — orkestrator ne sme runaway-uje).

### Phase 3 — telemetrija

**Deliverable:** A/B podaci za empirijsku odluku.

1. `pipeline-metrics.log` proširenje: nova kolona `reviewer_mode` (vrednosti `dispatched|orchestrator`). Schema bump dokumentovan u SPEC.md.
2. `apd_pipeline_metrics()` MCP tool (v6.2.0+) vraća polje u svakom run-u.
3. Novi `apd_reviewer_telemetry()` MCP tool koji agregira:
   - Last N runs (default 20) grupisanih po `reviewer_mode`
   - Srednji `adversarial_D` per mode
   - Standard deviation
   - Outlier flagovi (D > srednji + 2σ)
4. `pipeline-doctor` (CC) i `codex-doctor` nova sekcija "Reviewer mode telemetry":
   - Ako ima >= 5 runs u oba moda: prikaži srednji D + delta. Ako delta `orchestrator - dispatched > 1.5` → WARN ("inline reviewer pipeline-i imaju ~2 dodatna defekta u proseku — možda simulacija").
   - Manje od 5 runs: dim hint "nedovoljno podataka".

### Phase 4 — empirijska validacija (in-flight, ne blokira release)

**Cilj:** u realnim task-ovima (BambiProject + drugi) pokupiti 10-15 inline-mode pipeline-a + 10-15 dispatched-mode pipeline-a za isti repo/agent set.

Decision matrix posle 30 task-ova ukupno:
- `orchestrator_D - dispatched_D <= 0.5` → inline radi pošteno. Kandidat: flip default u v6.7 ili dodati `pipeline_mode: polish` automatski.
- `0.5 < delta <= 1.5` → marginalan. Ostavi opt-in, ne flipuj. Doc warning u skill.
- `delta > 1.5` → orkestrator simulira. Dva pravca:
  - (a) Pojačaj skill (više rigorous Citations, više hard-gates), retry.
  - (b) Auto-downgrade: po 3 uzastopna pipeline-a sa D > N, `pipeline-advance` blokira novi `reviewer: orchestrator` advance i forsira `dispatched`.

### Phase 5 — opcioni flip ili revert

Posle Phase 4 podataka (verovatno 4-6 nedelja real usage), odluka:

- **Flip default na orchestrator** u v6.7 — samo ako Phase 4 pokazuje delta ≤ 0.5 AND user posebno traži (memorija `feedback-version-bump-user-decision`).
- **Ostavi opt-in** — najverovatniji scenario. Inline kao "expert mode" za polish task-ove.
- **Revert** — ako Phase 4 dokazuje regresiju, ukloniti `apd-review` skill + reviewer grane iz `pipeline-advance` u v6.7. Telemetry kolona ostaje u logu kao istorijski podatak.

   - File:line referenca na nepostojeći fajl
   - LN > wc -l (line out of range)

## Granice scope-a (NE radimo u v6.6)

- Ne diramo dispatched reviewer logiku osim grananje na `reviewer: orchestrator`. Dispatched mod ostaje bit-for-bit isti.
- Ne menjamo adversarial-reviewer ni u jednom pogledu.
- Ne menjamo builder dispatch (inline builder ostaje Codex-only).
- Ne uvodimo "inline adversarial" — nema smisla, adversarial je anchor.
- Ne flipujemo default u v6.6. Phase 5 odluka tek u v6.7+.
- Nikakvo auto-conversion postojećih dispatched task-ova u inline mid-pipeline. Spec advance time decision.
