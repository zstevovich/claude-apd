# v6.15 — Pipeline Guide split: separate the tutor from the brainstorm

**Status:** IMPLEMENTED 2026-06-10 (test-codex-adapter 662/0; bump + commit pending user go-ahead). Bonus: test-system repaired (broken since v6.8.1/v6.8.11 — 15 FAIL on HEAD → 9 pre-existing env FAILs remain, all marker/plan fixture issues fixed)
**Type:** minor (new skill + gate swap)
**Enforcement impact:** floor unchanged — gate moves from one marker to another; education layer strengthened

## 1. Problem

`apd-brainstorm` carries two unrelated roles since v6.8.4:

1. **Clarification dialogue** (its original identity) — one-question-at-a-time scope
   convergence for vague tasks. Interactive, genuinely skippable when scope is
   pre-aligned.
2. **Pipeline tutor** (bolted on in v6.8.4) — downstream gates education
   (`**Implements:**` plan format, rationale file format, common BLOCKs + recovery,
   marker mechanics). Non-interactive, cheap (~2-3K tokens), and NEVER legitimately
   skippable — the orchestrator always needs the current gate contract.

The dual role leaks in practice:

- **Semantic confusion (root cause).** The skill is named "brainstorm" and its
  description promises clarification. An orchestrator with a perfectly clear task
  rationally concludes "I don't need brainstorming" — it has no way to know the
  skill also carries the pipeline operating manual. This is a mislabeled box, not
  orchestrator indiscipline.
- **The skip valve discards education along with ceremony.** Bambi 2026-06-09/10:
  8/8 runs skipped brainstorm with textbook-legitimate reasons — and thereby also
  skipped the tutor content. MONITORING #1 residual (~19% skip→single-block,
  typically a forgotten `Implements` header) is exactly education-shaped.
- **v6.8.8 TWO-PART CHECK is the patch that proves the disease.** "Scope clear"
  alone was not allowed to justify a skip; the orchestrator must also recite
  "APD config clear (...)". That second part is the guide function smuggled into a
  skip condition. Recent Bambi skip reasons all recite gate knowledge — but
  recitation is not loading: when a gate changes in a new plugin version, the
  recitation stays stale.
- **workflow.md cannot fill the role.** It is a per-project copy that drifts
  (hence v6.10 drift detection) and a passive SessionStart document (the entire
  v6.8 chain was a fight with that fact). A plugin-shipped skill is always in
  lockstep with the installed gate code and lands in context at task start —
  education at the moment it is needed, preventive rather than post-error
  (v6.8.3 lesson: BLOCK messages shorten recovery but don't prevent).

Cost framing (feedback-resource-cost-vs-quality): a per-task guide load is ~10×
cheaper than one BLOCK loop. Requiring it unconditionally is "force discipline at
entry", consistent with the v6.8.11 rationale.

## 2. Design decisions (converged)

| # | Decision | Choice |
|---|---|---|
| D1 | New skill name | `apd-pipeline-guide` — "guide" signals manual, "pipeline" says what for. Pushy description: "MANDATORY before every pipeline task — gates, formats, recovery." |
| D2 | Gate shape | `.guide-marker` is an **unconditional hard gate** on `apd pipeline spec` — **no skip valve**. Nothing interactive to skip; reading is cheaper than negotiating. No reason argument, no gaming surface. |
| D3 | Brainstorm role | Slims back to pure clarification (steps 1–4, R-criteria, Risks/Rollback, design summary). **Loses its gate role entirely.** No marker, no skip flag, no TWO-PART CHECK. Advisory skill that earns usage on merit. |
| D4 | brainstorm-marker | **Dies.** `--skip-brainstorm` flag, reason validation, INFO skip logging, TWO-PART CHECK — all removed. Net simplification of pipeline-advance: "guide-marker exists or BLOCK". |
| D5 | Marker scope | Per-task (matches pipeline state lifecycle; cheap). Reset wipes it like other pipeline state. |
| D6 | Lean mode | Guide required in Lean too — Lean tasks are precisely where skips happen. Single guide content; polish-mode specifics are already part of the gate table. |
| D7 | Trust model | Same as today's brainstorm-marker: marker is writable without loading the skill. The split raises EDUCATION, not enforcement; the floor stays on the existing guards. Honest framing, documented in SPEC. |

## 3. Deliverables

### 3.1 New skill `apd-pipeline-guide` (CC `skills/` + Codex `plugins/apd/skills/` mirror)

Content = the v6.8.4 tutor sections **extracted verbatim-then-polished** from
`apd-brainstorm/SKILL.md`:

- Pipeline phase map (spec → builder → reviewer → adversarial → verifier → commit)
  with what each gate checks — one-screen digest.
- Implementation plan format: `**Implements:** R<N>`/`none` on EVERY `### Section`,
  NO RESERVED NAMES rule, bidirectional check semantics (current §4b).
- Adversarial rationale format: `.adversarial-rationale.md` (`.md`!), per-finding
  `## Finding N` + Severity/Status/Rationale ≥40 chars, 100%-dismiss hard-block
  (current §4b second half).
- Common BLOCKs + recovery table (current §4c, minus deprecated max_defects rows —
  keep one line pointing at v6.9 deprecation, drop the prescriptive history).
- `pipeline_mode: polish` / Lean vs Full one-liner.
- Read-path instruction: inspect state via `apd pipeline show [spec|plan|state]`,
  never `cat`/`ls` on `.apd/pipeline/` (feeds the #4/#5 adoption problem from the
  2026-06-10 live-verify).
- Exit step: write `.guide-marker` (`<task-name>|<ISO-8601>`) — same format and
  write channel as today's brainstorm-marker.

Frontmatter: `effort: low` (read-and-acknowledge, not a dialogue), pushy
description per D1, `allowed-tools: Read` (it teaches; it doesn't explore).

### 3.2 `apd-brainstorm` redesign (both runtimes)

Remove: §"Default: load on every new task", TWO-PART CHECK, skip-reason mechanics,
§4b, §4c, marker write in §5, `--skip-brainstorm` examples, v6.8.x empirical
narration tied to gating. Keep: Iron Law, When to use/skip (scope-clarity criteria
only), Process steps 1–4, Risks/Rollback requirements, design summary template,
HARD-GATE (no code during brainstorm), Red Flags, Exit criteria. Hand-off becomes:
"after explicit approval → load `/apd-pipeline-guide` (mandatory gate), then write
spec-card.md and advance." Description frontmatter rewritten to clarification-only
triggers.

### 3.3 `pipeline-advance` gate swap

- Spec advance: require `$PIPELINE_DIR/.guide-marker` (name match + presence) —
  BLOCK message: "Load /apd-pipeline-guide first (writes .guide-marker)". New
  `log_block "guide-marker-missing"`.
- Delete: `--skip-brainstorm` arg parsing (v6.8.8), reason validation
  (v6.8.11), `brainstorm-skipped` INFO writer, brainstorm-marker check.
- **Deprecation shim:** a caller passing `--skip-brainstorm` gets a clear error:
  "flag removed in v6.15 — load /apd-pipeline-guide instead (no skip path)".
  One release of shim, removed in v7.0.
- Both reset paths + spec re-advance wipe `.guide-marker` (replace
  `.brainstorm-marker` in the two `rm -f` lists).
- Session-log reset filter: `brainstorm-skipped` INFO entries no longer produced;
  keep the case branch tolerant of historical entries (no parser break on old logs).

## 4. Telemetry / watch changes

- `skip→outcome` correlation (MONITORING #1/#2) ends — it answered its question
  (skips were legitimate, no cascades). Historical guard-audit entries remain
  parseable.
- **New watch:** does the orchestrator load `apd-brainstorm` voluntarily when scope
  is genuinely unclear, now that nothing forces it? Healthy signal = brainstorm
  load on vague tasks, absent on pre-scoped ones. Source: skill-load observability
  is limited; proxy = presence of design-summary convergence in session vs
  cold spec writes that hit quality BLOCKs.
- Guide-gate BLOCK rate (`guide-marker-missing`) expected to spike briefly per
  project after upgrade (one BLOCK → orchestrator learns), then ~0.

## 5. Migration

- Projects mid-task during upgrade: `.brainstorm-marker` exists, `.guide-marker`
  doesn't → next spec advance BLOCKs once with the new message; recovery = load
  guide (~30s). Acceptable; release notes call it out.
- Per-project `workflow.md` copies keep old instructions until re-init; drift
  dimension (3.6) flags them. apd-setup re-run refreshes.
- CHANGELOG migration section: flag removal (`--skip-brainstorm`), new mandatory
  skill, one-BLOCK upgrade bump.
