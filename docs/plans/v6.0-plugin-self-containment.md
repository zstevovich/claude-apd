# v6.0 — Plugin self-containment

> Pisano na kraju v5.0.11 sesije, izvršiti u sledećoj sesiji.

## Cilj

Premestiti sve plugin resurse iz repo root-a u `plugins/apd/` tako da
Codex plugin cache (`~/.codex/plugins/cache/codex-apd/apd/<v>/`) sadrži
sve što plugin treba — uključujući MCP server. Posle toga
plugin-shipped `.mcp.json` self-registration radi i `install-codex-config`
postaje cleanup-only.

Dva runtime-a (CC + Codex) i dalje žive u istom repo-u i instaliraju se
iz istog izvora — samo se path reference ažuriraju.

3. **Ako v5.0.11 nije pushovan: push pre nego što krećemo.**

## Plan migracije — fazno

### Phase 1 — file-system move

Premestiti unutar `plugins/apd/`:

```
bin/                → plugins/apd/bin/
mcp/                → plugins/apd/mcp/
rules/              → plugins/apd/rules/
hooks/              → plugins/apd/hooks/
templates/          → plugins/apd/templates/
VERSION             → plugins/apd/VERSION
monitors/           → plugins/apd/monitors/
```

Ostaje na repo root:

```
.claude-plugin/plugin.json    ← CC plugin manifest (CC marketplaces očekuju ovde)
.agents/plugins/marketplace.json  ← Codex marketplace
plugins/apd/                   ← sve plugin resurse unutra
README.md, CHANGELOG.md, LICENSE
docs/, examples/               ← GitHub-only (ne u plugin distribuciji)
bump-version, .gitignore       ← repo tooling
```

**Komanda za move (atomic, jedan commit):**

```bash
git mv bin plugins/apd/bin
git mv mcp plugins/apd/mcp
git mv rules plugins/apd/rules
git mv hooks plugins/apd/hooks
git mv templates plugins/apd/templates
git mv monitors plugins/apd/monitors
git mv VERSION plugins/apd/VERSION
```

### Phase 2 — path reference update

Sve skripte koje računaju `APD_PLUGIN_ROOT` iz `__file__` ili `$0` —
**ne moraju da se menjaju** (relativno samima sebi). Ali svaka skripta
koja referencira drugi resurs **kroz `APD_PLUGIN_ROOT`** treba reviziju:

- `bin/lib/resolve-project.sh` — verovatno ne dotiče repo strukture
- skripte koje rade `${APD_PLUGIN_ROOT}/templates/...` — sad to znači
  `plugins/apd/templates/`, ne `templates/`
- skripte koje rade `${APD_PLUGIN_ROOT}/rules/workflow.md` — sad
  `plugins/apd/rules/workflow.md`
- MCP server `apd_mcp_server.py:19` — `APD_PLUGIN_ROOT = parent.parent`
  će postati `plugins/apd/` (umesto repo root). Sve dalje reference
  rade isto jer su sada relativne plugin folderu.

**CC plugin manifest path reference:**

- `.claude-plugin/plugin.json` — hookovi referenciraju
  `${CLAUDE_PLUGIN_ROOT}/bin/...`. CC plugin root = repo root.
- Posle move-a, hookovi treba da idu na
  `${CLAUDE_PLUGIN_ROOT}/plugins/apd/bin/...`.
- Update svih hook entry-ja u `.claude-plugin/plugin.json`.

**Codex plugin manifest:**

- `plugins/apd/.codex-plugin/plugin.json` — `skills: "./skills/"` već
  relativno na plugin root (`plugins/apd/`), nema promene.
- Sad može da doda `"hooks": "./hooks/hooks.json"` ako želimo da Codex
  takođe učita hook-ove iz plugin manifest-a.

**`bump-version` skripta:**

- Update path-eva za `plugin.json`, `marketplace.json`, `VERSION` —
  VERSION se sada nalazi u `plugins/apd/VERSION`.
- Reference za update u README.md, CLAUDE.md, MEMORY.md, status.md
  ostaju iste (oni su na repo root-u).

### Phase 3 — `.mcp.json` self-registration

Sa svim resursima u plugin folderu, plugin .mcp.json sad može:

```json
{
  "mcpServers": {
    "apd": {
      "type": "stdio",
      "command": "uv",
      "args": ["run", "--with", "mcp", "python", "mcp/apd_mcp_server.py"],
      "cwd": ".",
      "tools": {
        "apd_ping":             { "approval_mode": "approve" },
        "apd_doctor":           { "approval_mode": "approve" },
        "apd_advance_pipeline": { "approval_mode": "approve" },
        "apd_guard_write":      { "approval_mode": "approve" },
        "apd_verify_step":      { "approval_mode": "approve" },
        "apd_adversarial_pass": { "approval_mode": "approve" },
        "apd_list_agents":      { "approval_mode": "approve" },
        "apd_pipeline_state":   { "approval_mode": "approve" }
      }
    }
  }
}
```

`cwd: "."` resolve-uje na `plugin_root` = `~/.codex/plugins/cache/codex-apd/apd/<v>/`,
gde sad postoji `mcp/apd_mcp_server.py`. Plus `tools.<name>.approval_mode`
nosi per-tool defaults — više nije potrebno pisati u user config.toml.

`install-codex-config` Block 1 postaje **cleanup-only** (briše legacy
[mcp_servers.apd*] blokove iz pre-v6.0 instala) — kao što je v5.0.10
pokušao i bilo bi da je radilo. Sada radi jer plugin cache ima fajl.

### Phase 4 — migration za postojeće user-e

User-i sa v5.0.11-and-earlier instala imaju `[mcp_servers.apd]` u svojim
`<project>/.codex/config.toml` sa apsolutnim path-om do njihovog dev
checkout-a (ako su radili `apd cdx init`) ili sa ničim (ako su čekali na
plugin to handle).

`install-codex-config` u v6.0 treba da:
1. Detektuje legacy `[mcp_servers.apd]` blok
2. Ako je apsolutni path ka starom `apd-template/mcp/`, ukloni blok
3. Ako je apsolutni path ka nečemu drugom (eksterna APD instalacija),
   warn ali ne diraj (ne overstep)

## Rollback strategija

Ako bilo šta od live testa fail-uje:

1. **Brzo unblock-uje korisnike** — `apd cdx init` u v6.0 treba da
   detektuje crash i ponudi fallback (vrati `[mcp_servers.apd]` sa
   apsolutnim path-om).
2. **Ako struktura nije održiva** — branch revert. Pošto je ovo major
   bump, radi se na zasebnom feature branch-u (`v6.0-self-containment`),
   ne direktno na main. Merge tek posle live verifikacije.

## Open questions za session start

1. **CC `${CLAUDE_PLUGIN_ROOT}` ostaje na repo root ili se premešta u
   `plugins/apd/`?**
   - Opcija A (preferred): ostaje na repo root, hooks reference
     `${CLAUDE_PLUGIN_ROOT}/plugins/apd/bin/...`. Manja promena.
   - Opcija B: premestiti `.claude-plugin/plugin.json` u
     `plugins/apd/.claude-plugin/plugin.json`. Veća, ali unifikuje sa
     Codex.
2. **Da li `examples/` kopija treba da se ažurira?** Verovatno da —
   `examples/nodejs-react/.claude/agents/*.md` možda referenciraju
   `${CLAUDE_PLUGIN_ROOT}/bin/...` — update na novi put.
3. **Backward-compat shim:** treba li simlinkovi na repo root-u
   (`bin/` → `plugins/apd/bin/`) za period tranzicije? Pomaže
   automation koja imje hardcoded `apd-template/bin/...` paths. Ali
   stvara potencijalno pitanje "isti fajl iz dva mesta".
4. **Verzija major bump (6.0.0)** — potvrditi sa userom pre commit-a.

  GitHub-only, ne idu u plugin cache, ne treba da budu u plugin folder.
- Zajednička `tests/` direktorijum — za sad ostaje gdegod je.
