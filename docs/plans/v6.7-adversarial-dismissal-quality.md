## Ključna dizajn odluka (iz 2026-05-18 diskusije)

### Format: Markdown, ne JSON

- `.apd/pipeline/` već je markdown-first (`spec-card.md`, `implementation-plan.md`, planiran `.reviewer-trace`). JSON bi bio strani element.
- Bash + `grep -c "^## Finding"` + `grep -c "^\*\*Status:\*\* dismissed"` daje sve što treba za gate logiku. Ako rationale-length check kasnije postane gnjavator → escalate na Go binary u v6.8 (YAGNI).
- Markdown forsira orkestratora da PIŠE rečenice. To je samo po sebi friction protiv "1 Critical + 3 Important. Dispatch fixes." pattern-a. JSON-ov `"rationale": "ok"` je 4 karaktera; markdown rationale ispod jednog reda izgleda vidljivo manjkavo.

### Klasifikacija: tri statusa, summary linija ostaje T:A:D

```
accepted                    — orkestrator dispatch-uje fix
dismissed                   — orkestrator svesno odbacuje (zahteva rationale)
reviewer-self-dismissed     — adversarial reviewer već dao razlog inline
```

Ključni potez: `D` u `.adversarial-summary` (`ADVERSARIAL:T:A:D`) ostaje **sum of orchestrator-dismissed + reviewer-self-dismissed**.

Razlozi za backward compat:
- `pipeline-metrics.log` već loguje T:A:D — kolone bi se polomile.
- `apd report`, session-log generator (`feedback_orchestrator_failures_2026_05_11`-style), Miro dashboard svi parsiraju 3-column format.
- Migration ka 4-column format-u bi bio v7 breaking change. Nepotrebno.

Umesto toga: verifier čita `.adversarial-rationale.md` i gate-uje na **dismissed-by-orchestrator (`Do`) count** (derived from rationale file), ne na summary `D`. Summary ostaje za historical compat.

### Gate signature

```
Hard gate:  T >= 3 && A == 0 && Do >= 1
            → BLOCK with explanation + path to write rationale file

Soft warn:  any rationale text < 40 chars
            OR matches lazy pattern: ^(ok|n/a|false positive|not applicable|skip|no)\.?$
            → emit warn in apd-finish output, NOT block

Structural: .adversarial-rationale.md must exist
            AND total Finding count must equal T from summary
            → BLOCK if missing or count mismatch
```

Reviewer-self-dismissed nikad **ne** triggers hard gate. Razlog: čisto reviewer-self-dismiss run (npr. T=5, A=0, Dr=5) je **zdrav adversarial run** — reviewer je radio svoj posao i počistio noise inline. Original v6.7 `T>=3 && A==0` gate bi to false-block-ovao. Trostuka klasifikacija to izbegava.

### MR.9 retroaktivna validacija dizajna

## File format spec — `.adversarial-rationale.md`

```markdown
# Adversarial Rationale — <task-name>
# Generated: <ISO-8601 timestamp>
# Findings: T=N (A=N accepted, Do=N orchestrator-dismissed, Dr=N reviewer-self-dismissed)

## Finding 1 — <one-line title>
**Severity:** critical | important | minor
**Status:** accepted | dismissed | reviewer-self-dismissed
**Rationale:** <≥40 chars; required for dismissed and reviewer-self-dismissed>

## Finding 2 — <one-line title>
**Severity:** ...
**Status:** ...
**Rationale:** ...

...
```

### Strict parser rules (`pipeline-advance verifier`)

1. **File exists.** Path: `$PIPELINE_DIR/.adversarial-rationale.md`. Missing → BLOCK.
2. **Header counts match summary.** `# Findings: T=8 (...)` header → T must equal column 1 of `.adversarial-summary`. Mismatch → BLOCK.
3. **One `## Finding N` block per finding.** Count blocks via `grep -c '^## Finding '`. Must equal T. Mismatch → BLOCK.
4. **Each block has all three required fields.**
   - `^\*\*Severity:\*\* (critical|important|minor)$`
   - `^\*\*Status:\*\* (accepted|dismissed|reviewer-self-dismissed)$`
   - `^\*\*Rationale:\*\* .+` (non-empty)
   Missing/malformed → BLOCK with specific finding number.
5. **Status accounting matches summary D.** `Do + Dr` (count of `dismissed` + `reviewer-self-dismissed` Status lines) must equal `D` from summary. Mismatch → BLOCK ("rationale statuses don't match summary").
6. **Rationale length.** For each dismissed and reviewer-self-dismissed entry, Rationale value must be ≥40 chars. Below → soft WARN (not block), surfaces in `apd-finish` output.
7. **Lazy pattern match.** Rationale matching `^(ok|n/a|false positive|not applicable|skip|no)\.?$` (case-insensitive, trimmed) → soft WARN.
8. **Hard gate.** If `T >= 3 && A == 0 && Do >= 1` → BLOCK regardless of rationale quality. This is the bypass-detection arm.

## Plan migracije — fazno

### Phase 1 — Verifier gate (MVP)

**Deliverable:** verifier čita `.adversarial-rationale.md`, hard-blocks na missing/malformed/100%-orch-dismiss. Soft warn-ovi nisu još implementirani.

1. `plugins/apd/bin/core/pipeline-advance` — verifier branch (current `.adversarial-summary` parse path):
   - Posle postojećeg `ADV_T/ADV_A/ADV_D` parse-a, dodati `parse_rationale_file()` bash function.
   - Function:
     - Check file exists at `$PIPELINE_DIR/.adversarial-rationale.md`. Missing → BLOCK message v7.1 below.
     - Parse `# Findings: T=N (...)` header. T must equal `ADV_T`. Mismatch → BLOCK v7.2.
     - Count `^## Finding ` blocks. Must equal `ADV_T`. Mismatch → BLOCK v7.3.
     - Per block: extract Severity, Status, Rationale via `awk` slice between `^## Finding ` markers. Missing field → BLOCK v7.4 with finding index.
     - Compute `Do = grep -c '^**Status:** dismissed$' file`, `Dr = grep -c '^**Status:** reviewer-self-dismissed$' file`.
     - `Do + Dr` must equal `ADV_D`. Mismatch → BLOCK v7.5.
   - Posle parsea: hard gate check: `[ "$ADV_T" -ge 3 ] && [ "$ADV_A" -eq 0 ] && [ "$Do" -ge 1 ]` → BLOCK v7.6.
   - Ako sve prođe: continue postojećim SIGN path-om.

2. **BLOCK messages** (kandidat tekst, refine prilikom implementacije):
   ```
   v7.1: BLOCKED: Adversarial rationale file missing.
         Write per-finding rationales to .apd/pipeline/.adversarial-rationale.md
         (format: ## Finding N + **Severity:** + **Status:** + **Rationale:**)
         Then re-run: bash .claude/bin/apd pipeline verifier

   v7.2: BLOCKED: Rationale file findings count mismatch.
         Header declares T=<N> but .adversarial-summary shows T=<M>.

   v7.3: BLOCKED: Rationale finding block count mismatch.
         Found <N> "## Finding" blocks but expected <M> per summary.

   v7.4: BLOCKED: Finding <N> missing required field: <Severity|Status|Rationale>.

   v7.5: BLOCKED: Rationale statuses don't match summary dismissed count.
         dismissed=<Do>, reviewer-self-dismissed=<Dr>, sum=<S>, summary D=<D>.

   v7.6: BLOCKED: 100% orchestrator dismissal pattern detected.
         T=<N> findings, A=0 accepted, Do=<N> orchestrator-dismissed.
         At least one finding must be accepted, or rationale must reclassify
         to reviewer-self-dismissed where appropriate. Re-write rationale and
         dispatch builder for accepted findings.
   ```

3. **Reset wipes rationale file.** `pipeline-advance reset` already wipes `.adversarial-summary`, `.adversarial-pending`. Add `.adversarial-rationale.md` to reset cleanup list.

4. **No CC/Codex split needed.** `pipeline-advance` is the shared backbone for both runtimes — single source.

### Phase 2 — Quality warnings (soft)

**Deliverable:** rationale-length + lazy-pattern checks emit `! WARN` lines in verifier output (and `apd-finish` summary), no block.

1. `parse_rationale_file()` extends:
   - Per dismissed/reviewer-self-dismissed entry, measure Rationale text length (`wc -m` after stripping `**Rationale:** ` prefix).
   - If < 40 chars → emit `! WARN Finding <N>: rationale text only <X> chars (≥40 recommended)`.
   - Lazy-pattern grep: `grep -iE '^\*\*Rationale:\*\* +(ok|n/a|false positive|not applicable|skip|no)\.?\s*$'` → per-match emit `! WARN Finding <N>: rationale matches lazy pattern`.

2. WARN lines go to verifier stdout AND get aggregated in `apd-finish` closure output:
   - `apd-finish` already reads `.adversarial-summary` for summary; add reading `.adversarial-rationale.md` quality flags.
   - Format: `! Adversarial rationale quality: <N> warnings (see verifier output)`.

3. WARN counts go to `pipeline-metrics.log` as new column? **DEFER do Phase 3** — Phase 2 keeps log format unchanged, only stdout warnings.

### Phase 3 — Adversarial-reviewer dispatch prompt update

**Deliverable:** adversarial-reviewer izlaz je strukturisaniji, eksplicitno deklariše per-finding status. Orkestrator više ne mora "intuitivno" da klasifikuje.

1. `plugins/apd/templates/dispatch/adversarial-reviewer.md` dispatch template update:
   - Postojeći output format (markdown findings list) zadrži.
   - Dodaj eksplicitan **Status:** linija per finding:
     ```
     1. **<title>**
        File: ...
        Severity: critical|important|minor
        Status: active | self-dismissed
        Note (if self-dismissed): <one-line reason>
        <detailed analysis>
     ```
   - `Status: active` znači "ovde je real defect, orkestrator mora doneti accepted/dismissed odluku".
   - `Status: self-dismissed` znači "reviewer je sam tokom analize zaključio da nije actionable; reason u Note polju".

2. Orkestrator workflow.md instrukcija update:
   - "Posle adversarial-reviewer-a, copy-paste finding-e u `.adversarial-rationale.md`. Za **active** finding, dodaj `**Status:** accepted` ili `**Status:** dismissed` + own rationale. Za **self-dismissed**, copy reviewer-ov Note kao Rationale i set `**Status:** reviewer-self-dismissed`."
   - Cross-runtime: workflow.md (CC) + AGENTS.md (Codex) oba update-ovati.

3. **Skill update — apd-finish:**
   - Anti-patterns sekcija: dodati "Skipping rationale file: write `.adversarial-rationale.md` BEFORE dispatching post-adversarial builder. Verifier will hard-block otherwise."
   - When-to-use: "Posle adversarial, pre nego što runeš builder #N+1 za fix-ove."

## Open decisions (debata pre Phase 1 implementacije)

1. **Rationale min length: 40 chars ili više/manje?** 40 je predlog. Premali (npr. 20) hvata samo "ok" — premalo signala. Preveliki (npr. 80) pravi friction. **Predlog: krenuti sa 40, pratiti u prvih 5-10 live pipelines, podizati ako se vidi 40-char gaming.**

2. **Hard gate threshold T>=3 ili T>=2?** Backlog memory says 3. Razlog: T=1 ili T=2 sa A=0 može legitimno biti out-of-scope flagging od strane reviewer-a. T>=3 sa A=0 je gde se pattern javlja u empirici. **Zadržati 3.**

3. **Lazy-pattern regex completeness.** Trenutni: `^(ok|n/a|false positive|not applicable|skip|no)\.?$`. Da li dodati: `looks good`, `acceptable`, `fine`, `existing pattern`, `out of scope`? Poslednja dva su tricky — mogu biti legitimne kratke rationale-e. **Predlog: krenuti sa minimum setom, dodavati po empirici.**

4. **Counter-rule `adversarial: enforcement=light`** iz postojeće memory: da li je još potreban? Backlog ga predlaže za "exploratory phase" projekte. **Predlog: implementirati kao opt-out polje u spec-card (`adversarial: rationale_gate=off`); default on. Identičan pattern kao postojeći `adversarial: max_defects` field iz v6.1 B2.**

5. **Phase 4 inclusion u v6.7 ili defer u v6.7.1?** Phase 4 dira pipeline-metrics log format — moguć ripple kroz `apd report`, `apd-miro`. **Predlog: defer u v6.7.1 patch, ship v6.7 sa fazama 1-3 (gate + quality warns + dispatch prompt update).**
