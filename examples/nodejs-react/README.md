# Example: Node.js + React Project

This directory shows what a fully configured APD environment looks like for a **Node.js + Express + React + PostgreSQL** project called "TaskFlow".

## What is included

| File | Purpose |
|------|---------|
| `CLAUDE.md` | Fully populated project instructions |
| `.claude/agents/backend-builder.md` | Backend agent with scope and hooks |
| `.claude/agents/frontend-builder.md` | Frontend agent with scope and hooks |
| `.claude/memory/MEMORY.md` | Populated memory index |
| `.claude/memory/status.md` | Current project status |

## What is NOT included

- **Scripts** — these are universal and live in the plugin (`${CLAUDE_PLUGIN_ROOT}/scripts/`). Only `verify-all.sh` lives in your project.
- **settings.json** — plugin hooks are in `hooks/hooks.json` within the plugin. Project-level settings are generated by `/apd-setup`.
- **Rules and skills** — universal, provided by the plugin.

## How to use this example

Compare these files against the template originals to understand what `/apd-setup` replaces:
- Every `{{PLACEHOLDER}}` is filled with concrete values
- Agents have real scope paths and correct hook configurations
- Memory files reflect an active project state
